# ADR-001: Vite 개발 서버 프록시 채택

> 작성일: 2026-07-10  
> 상태: 확정

## 컨텍스트

Admin 대시보드에서 개발(Dev)과 프로덕션(Prod) 서버를 동시에 탭으로 관리해야 합니다.  
로컬 개발 환경에서 외부 백엔드 서버로 직접 요청 시 CORS 문제가 발생합니다.

---

## 검토 대상

| 방법 | 설명 | 문제점 |
|------|------|--------|
| 백엔드 CORS 허용 | `Access-Control-Allow-Origin: *` | 관리자 API 엔드포인트 노출 |
| 별도 Nginx 프록시 | 중간 프록시 서버 운영 | 로컬 개발에 인프라 추가 필요 |
| **Vite 개발 서버 프록시** | `vite.config.js` proxy 설정 | 개발 환경에서만 동작, 빌드 결과물에 미포함 |

---

## 결정: Vite 개발 서버 이중 프록시 채택

### 이유

1. **백엔드 수정 불필요**: 서버 CORS 설정 변경 없이 로컬에서 외부 API에 접근할 수 있습니다.

2. **두 환경 동시 접근**: `/proxy-dev`와 `/proxy-prod` 경로로 각각 Dev/Prod 서버를 분기하여 하나의 대시보드에서 양쪽 환경을 관리할 수 있습니다.

3. **빌드 결과물에 미포함**: Vite 프록시는 개발 서버에서만 동작하며, Vercel 배포 시에는 직접 URL을 사용하므로 보안상 안전합니다.

### 트레이드오프 (수용 가능)

- 로컬 환경에서만 프록시가 동작하므로 Vercel 배포 시 환경 변수(`VITE_PROXY_DEV_TARGET`, `VITE_PROXY_PROD_TARGET`)를 별도 설정 필요
- Vite에 종속되어 빌드 툴 변경 시 프록시 설정 재작성 필요
