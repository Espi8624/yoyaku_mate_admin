# ADR-001: Vite 개발 서버 프록시 채택

> 작성일: 2026-07-10  
> 상태: 개정됨 (Revised / 빌드 모드별 단일 바인딩으로 개편)

## 컨텍스트

Admin 대시보드에서 외부 백엔드 서버로 직접 요청 시 CORS 문제가 발생합니다.
과거에는 UI의 개발(Dev)/본방(Prod) 탭을 통해 두 환경에 동시 접근할 수 있도록 이중 프록시 설정을 채택했으나, 이로 인해 로컬 개발 시 DB와 백엔드가 명확히 구분되지 않고 혼선을 유발하여 **실행/빌드 모드별 단일 바인딩 방식**으로 개정되었습니다.

---

## 결정: Vite 개발 서버 단일 프록시 채택

### 2026-07-10 개정 사항

UI 상의 탭 변경에 따른 동적 API 주소 전환 방식을 제거하고, `npm run dev:dev` 및 `npm run dev:prod`와 같이 사용자가 구동하려는 실행 환경(`.env`) 파일에 맞춰 자동으로 단일 백엔드에 연동되도록 프록시를 일원화했습니다.

1. **안전성 확보**: 빌드/구동 시점에 바라볼 DB/백엔드 타겟이 지정되므로, 개발 중에 실수로 운영 API를 호출하거나 잘못된 DB 커넥션으로 요청을 쏘는 대형 사고를 방지합니다.

2. **Vite 단일 프록시 통합**: `/proxy-dev`, `/proxy-prod` 대신 `/api/admin` 단일 엔드포인트로 프록시 설정을 간소화했습니다.

3. **CORS 우회 유지**: 로컬 개발 시 Vite 프록시 서버를 통해 외부 API를 호출함으로써 CORS 문제를 여전히 안전하게 회피합니다.
