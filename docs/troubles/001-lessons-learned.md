# 開発プロセスでの学び (Lessons Learned)

> 作成日: 2026-07-10

---

## MUI カスタムテーマの構築によるデザインの一貫性確保

MUI v7 のテーマシステムをチューニングし、共通のプライマリーカラー設定やタイポグラフィの規則を定義しました。  
これにより、今後新しい画面や設定項目などの管理用UIコンポーネントを追加する際、個別に細かくスタイルを調整する手間が省け、画面間の一貫性を容易に保てるようになりました。

---

## Custom Hook の導入による API 処理の分離

各コンポーネント内に散らばりがちな通信状態管理(`isLoading`, `error`)やAxiosリクエスト処理をカスタムフックとして隠蔽・集約しました。  
これにより、コンポーネント側はUI表現(View)に集中し、データの取得や変更処理(Controller)を美しく分離することができました。

---

## 事業者登録証画像の一時アクセスURL方式

管理画面上でデリケートな個人情報を含む事業者登録証の画像を安全に表示するため、画像を一般公開バケットに直接アップロードして公開するのではなく、DBには一意の暗号化キー名(UUID)のみを保存し、表示のたびに一時アクセス署名付きURLを都度発行(5分間有効)する仕組みを適用しました。  
これにより、悪意ある第三者への情報流出のリスクを最小限に抑えています。

---

## Vite 7 ビルドパイプラインの最適化

Vite 7の超高速なローカル開発時HMR(Hot Module Replacement)の恩恵を最大化しつつ、本番配備時にはチャンクの分割等の最適化を行うことで、ダッシュボード画面の初回ロード時間の高速化とVercel上での配備後の動作安定性を実現しました。
