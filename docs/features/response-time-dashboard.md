# 機能仕様書: Response Timeダッシュボード (Response Time Dashboard)

本文書は、`yoyaku_mate_admin` 本社管理者ウェブに実装されたAPIレイテンシリアルタイムモニタリング機能の仕様を説明します。

> 作成日: 2026-07-20  
> 関連文書: [実装詳細書: Response Timeダッシュボード](../implementation/response-time-dashboard.md)

---

## 1. 概要 (Overview)

本社管理者がサーバーAPIのレスポンス速度をリアルタイムで監視し、遅いエンドポイントを即座に特定できるようにするパフォーマンスモニタリング画面です。

単純な平均レイテンシの表示にとどまらず、P95 / P99パーセンタイル指標とエンドポイント別の詳細統計を提供することで、**実際の本番環境でのボトルネックを正確に特定**できるよう設計されています。

---

## 2. 画面構成および主要指標 (UI Metrics)

管理者バックオフィスの専用ダッシュボードとして表示され、5秒周期で自動更新（HTTP Polling）されます。

### 2.1 時間範囲選択タブ (Time Range Toggle)

画面上部のトグルタブで集計基準時間を切り替えます。タブ変更時に即座にデータが更新されます。

| タブ | 集計範囲 | 主な用途 |
|---|---|---|
| **直近5分** | 直近5分 | リアルタイム障害検知 |
| **直近1時間** | 直近1時間 | 一般運用モニタリング（デフォルト） |
| **直近24時間** | 直近24時間 | 日別パフォーマンス推移確認 |

### 2.2 上部サマリーカード (Summary Cards)

選択された時間範囲内の全リクエストに対する集約指標を4枚のカードで表示します。

| カード | 説明 | 色アラート基準 |
|---|---|---|
| **AVG LATENCY** | 全体平均レスポンス時間 (ms) | < 100ms 緑 / 100~499ms 橙 / ≥ 500ms 赤 |
| **P95 LATENCY** | 上位5%リクエストのレイテンシカットライン | 同上 |
| **P99 LATENCY** | 上位1%リクエストのレイテンシカットライン | 同上 |
| **ERROR RATE** | 4xx / 5xx エラーリクエスト比率 (%) | ≤ 1% 緑 / 1~5% 橙 / > 5% 赤 |

### 2.3 下部エンドポイント別レイテンシテーブル (Endpoint Latency Table)

選択時間範囲内で、平均レスポンス時間基準の降順で上位10エンドポイントを表示します。

* **グリッド列**:
  * **METHOD**: HTTPメソッドバッジ (GET / POST / PATCH / DELETE、メソッド別に色分け)
  * **ENDPOINT**: APIパス (monoスペースフォント)
  * **AVG**: 平均レスポンス時間 (ms)、閾値別の動的な色
  * **P95**: 95th Percentileレイテンシ、閾値別の動的な色
  * **P99**: 99th Percentileレイテンシ、閾値別の動的な色
  * **REQUESTS**: 当該期間内の総呼び出し回数
  * **ERROR %**: 当該エンドポイントのエラーリクエスト比率、閾値別の動的な色
