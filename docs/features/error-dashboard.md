# エラーダッシュボードUI仕様 (Error Dashboard UI)

> 最終更新: 2026-07-11

## 概要

本社の管理者がシステム全体の稼働状態を瞬時に把握できるよう、サーバー内で発生したエラー統計および詳細な履歴をリアルタイムで監視・追跡するバックオフィス用ダッシュボード画面の仕様です。  
[ErrorCountPage.jsx](../../src/pages/ErrorCountPage.jsx) コンポーネントをベースに実装されています。

---

## 画面レイアウト設計

ダッシュボード画面は大きく2つのセクションで構成され、いずれもレスポンシブWebデザインに対応しています。

```
+-----------------------------------------------------------+
|  [!] Error Dashboard                                      |
|  リアルタイムエラー監視およびトラッキングシステムの...       |
|                                                           |
|  +--------------+ +--------------+ +--------------+ +-------...  |
|  | 500 ERRORS   | | 400 ERRORS   | | DB ERRORS    | | SSE D...  |
|  |     0        | |     6        | |     0        | |     0     |
|  +--------------+ +--------------+ +--------------+ +-------...  |
|                                                           |
|  最近のエラーログ                                           |
|  +-----------------------------------------------------+  |
|  | 日時        | タイプ        | API経路       | メッセ...|  |
|  +-------------+---------------+---------------+-------|  |
|  | 2026/07/10  | 400_BAD_RE... | GET /api/...  | Not...|  |
|  +-----------------------------------------------------+  |
+-----------------------------------------------------------+
```

### 1. メトリクスサマリーカード (上部)
* **構成**: 4つの集計値カードが横並びに配置されます。
* **表示項目**:
  * **500 ERRORS**: サーバー内部の深刻なシステムエラー（Go Panic、ランタイムエラーなど）の合計値。
  * **400 ERRORS**: クライアントから送信された不正なリクエストや404エラーの合計値。
  * **DB ERRORS**: MongoDBのクエリ失敗やコネクション切断など、DB層でのエラー合計値。
  * **SSE DISCONNECTS**: 待機列の配信中に発生したクライアント接続切断の合計値。
* **配置ルール**: MUI Grid v2の最新仕様を適用し、大画面では1列に4つが均等に並び、画面幅に合わせて自動的に折り返されます。

### 2. 最近のエラーログテーブル (下部)
* **構成**: 直近50件のエラー履歴を表示するデータグリッドです。
* **表示カラム**:
  * **日時**: エラー発生時刻（日本標準時に自動変換して表示）。
  * **タイプ**: 400/500/DB/SSEエラーの識別ラベル。
  * **API経路**: エラーが検出されたHTTPメソッドとエンドポイントパス（例: `GET /api/admin/metrics`）。
  * **メッセージ概要**: エラーメッセージの要約（長い場合は省略表示）。
* **詳細表示**: テーブル内の行をクリックすると、詳細なコールスタックや詳細ログを確認できるモーダルウィンドウがポップアップ表示されます。

## 技術実装詳細

### 1. MUI Grid v2 レスポンシブ設計の適用
Material UIの旧仕様（`item` や `xs`, `md`, `lg` プロップ）の廃止に伴い、新しいGrid v2規格に完全対応させています。
```jsx
// 適用コード例 (ErrorCountPage.jsx)
<Grid container spacing={3} sx={{ mb: 4 }}>
  <Grid size={{ xs: 12, md: 6, lg: 3 }}>
    <Card>...</Card>
  </Grid>
</Grid>
```
* `size={{ xs: 12, md: 6, lg: 3 }}` を指定することで、モバイル（`xs`）では幅100%（縦に1列）、タブレット（`md`）では50%（2列）、デスクトップ（`lg`）では25%（4列）として、他のダッシュボード画面と完璧に横幅と高さが揃うよう調整されています。

### 2. 5秒間隔の自動更新 (Polling)
ダッシュボードがバックグラウンドで開きっぱなしになっている間、最新情報を常に維持するためにReactの `useEffect` を用いて5秒周期で非同期通信を行います。
```javascript
useEffect(() => {
  fetchData();
  const interval = setInterval(fetchData, 5000); // 5秒周期で自動更新
  return () => clearInterval(interval);
}, []);
```

### 3. API通信とレスポンスデータの展開
バックエンドのレスポンス封筒フォーマット（Envelope Pattern）を考慮し、[adminService.js](../../src/api/adminService.js)を介して安全にデータを取り出します。
```javascript
// adminService.js でのラッパー解除処理
export const getErrorMetrics = async () => {
  const response = await apiClient.get('/metrics/errors');
  // response.data?.data (成功時のラップ解除) または response.data (フォールバック)
  return response.data?.data || response.data;
};
```
これにより、受け取ったオブジェクト構造が直接React状態管理（`setMetrics`、`setLogs`）に流し込まれ、UIコンポーネントが破綻せずに最新の数字を描画できます。
