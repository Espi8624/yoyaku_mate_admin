# 에러 대시보드 UI 사양 (Error Dashboard UI)

> 최종 수정: 2026-07-11

## 개요

본사 관리자가 플랫폼 전체의 시스템 안정성 및 장애 상태를 즉각적으로 인지할 수 있도록, 서버 내부 오류 통계와 에러 발생 이력을 실시간으로 감시하고 트래킹하는 본사 관리자 백오피스용 대시보드 화면 사양입니다.  
[ErrorCountPage.jsx](../../src/pages/ErrorCountPage.jsx) 컴포넌트를 기반으로 설계 및 구현되었습니다.

---

## 화면 레이아웃 설계

화면은 크게 상단의 **에러 통계 카드** 영역과 하단의 **최근 상세 로그 테이블** 영역으로 나누어 구성되어 있으며, 반응형 웹 디자인을 지원합니다.

```
+-----------------------------------------------------------+
|  [!] Error Dashboard                                      |
|  실시간 에러 감시 및 트래킹 시스템 대시보드입니다.           |
|                                                           |
|  +--------------+ +--------------+ +--------------+ +-------...  |
|  | 500 ERRORS   | | 400 ERRORS   | | DB ERRORS    | | SSE D...  |
|  |     0        | |     6        | |     0        | |     0     |
|  +--------------+ +--------------+ +--------------+ +-------...  |
|                                                           |
|  최근 에러 로그                                             |
|  +-----------------------------------------------------+  |
|  | 일시        | 타입          | API경로       | 메시지...|  |
|  +-------------+---------------+---------------+-------|  |
|  | 2026/07/10  | 400_BAD_RE... | GET /api/...  | Not...|  |
|  +-----------------------------------------------------+  |
+-----------------------------------------------------------+
```

### 1. 에러 메트릭 요약 카드 (상단)
* **구성**: 총 4가지 에러 카테고리에 대한 누적 집계값을 가로 그리드로 보여줍니다.
* **표시 메트릭**:
  * **500 ERRORS**: 고 내부 패닉, 컴파일 런타임 등 500 계열 서버 내부 실패 횟수
  * **400 ERRORS**: 잘못된 요청 인수 전송, 존재하지 않는 404 경로 호출 등 400 계열 실패 횟수
  * **DB ERRORS**: MongoDB 접속 유실, 쿼리 구문 예외 등 데이터베이스 영역 에러 발생 횟수
  * **SSE DISCONNECTS**: 대기열 실시간 중계 중 브라우저 탭 닫기 등으로 강제 유실된 SSE 연결 횟수
* **배치 규칙**: MUI Grid v2 최신 명세를 적용하여, 데스크톱 화면 기준 1줄에 4개의 카드가 25%씩 영역을 균등 분할해 정렬됩니다.

### 2. 최근 에러 로그 테이블 (하단)
* **구성**: 가장 최근에 백엔드 DB에 적재된 에러 로그 데이터 최대 50건을 최신 정렬로 노출합니다.
* **그리드 컬럼**:
  * **일시**: 에러 발생 시각 (사용자의 로컬 디바이스 타임에 맞춰 자동 변환)
  * **타입**: 에러의 대분류 식별 코드 (예: `400_BAD_REQUEST`)
  * **API경로**: 호출을 유발한 API 메소드 및 경로 (예: `GET /api/admin/metrics`)
  * **메시지개요**: 발생 에러의 요약 메시지 (지정 길이를 초과할 경우 생략 부호 처리)
* **상세 팝업**: 테이블 행을 마우스로 클릭하면, 문제 해결을 돕는 에러 상세 스택 트레이스 등을 별도의 레이어 모달창으로 팝업 제공합니다.

## 기술 구현 상세

### 1. MUI Grid v2 반응형 레이아웃 마이그레이션
Material UI의 기존 레거시 프로퍼티인 `item` 키워드와 `xs`, `md`, `lg` 단일 속성이 더 이상 작동하지 않는 스펙 변경에 대응하여 Grid v2의 `size` 프로퍼티를 완벽하게 도입했습니다.
```jsx
// 마이그레이션 적용 코드 (ErrorCountPage.jsx)
<Grid container spacing={3} sx={{ mb: 4 }}>
  <Grid size={{ xs: 12, md: 6, lg: 3 }}>
    <Card>...</Card>
  </Grid>
</Grid>
```
* `size={{ xs: 12, md: 6, lg: 3 }}` 구성을 통해 모바일에서는 가로 폭 100%(1단 배열), 태블릿에서는 50%(2단 배열), 대형 모니터에서는 25%(4단 배열)로 가로 너비와 카드 높이가 다른 대시보드 화면들과 통일성을 가지며 완벽히 가로폭에 안착하도록 처리되었습니다.

### 2. 5초 실시간 데이터 폴링 (Polling)
어드민 모니터링 화면을 브라우저에 띄워 둔 동안 실시간에 가깝게 상태를 동기화하기 위해 React `useEffect` 훅을 활용하여 5초 주기로 자동 갱신 조작을 수행합니다.
```javascript
useEffect(() => {
  fetchData();
  const interval = setInterval(fetchData, 5000); // 5초 주기로 데이터 백그라운드 fetch
  return () => clearInterval(interval);
}, []);
```

### 3. API 응답 봉투(Envelope) 해제 처리
백엔드 API가 보내오는 공통 규격인 `{ status: "success", data: ... }` 형태를 안전하게 풀어서 React 상태에 입력하기 위해 [adminService.js](../../src/api/adminService.js) 계층에서 가공 가이드를 추가하여 연동했습니다.
```javascript
export const getErrorMetrics = async () => {
  const response = await apiClient.get('/metrics/errors');
  // API의 봉투 포맷을 벗겨서 데이터 영역만 추출
  return response.data?.data || response.data;
};
```
이 래퍼 무력화 처리를 통해 화면상의 에러 카운터 상태(`metrics`)와 로그 목록 상태(`logs`)가 0이 아닌 백엔드 실제 데이터 값으로 완벽하게 파싱되어 그려집니다.
