# 機能仕様書: システムメトリクスダッシュボード (System Metrics Dashboard)

本ドキュメントは、管理者用画面（`yoyaku_mate_admin`）の**システムメトリクス (System Metrics)** 機能に関するフロントエンドの要件および仕様を定義します。

> 作成日: 2026-07-23  
> 対象: `yoyaku_mate_admin` (React UI)

---

## 1. 概要 (Overview)
管理者がバックエンドサーバー（fly.ioインスタンス）の実際のハードウェアリソース（CPU、Memory、Disk）の状態をリアルタイムでモニタリングするためのページです。サーバーの障害を事前に検知し、直感的なインフォグラフィックで状態を把握できるように支援します。

## 2. 主な機能および要件 (Key Features & Requirements)

### 2.1 ハードウェアリソース指標の視覚化
- **CPU USAGE**: サーバー全体のCPU使用率 (%)
- **MEMORY USAGE**: サーバー全体の仮想メモリ使用率 (%)
- **DISK SPACE**: ルート(`/`)パーティションのディスク使用率 (%)
- 上記3つの指標をカード（Card）コンポーネントとして並べて配置し、一目で把握できるようにします。

### 2.2 5秒周期のリアルタイム自動更新 (Polling)
- WebSocketの代わりに、実装の複雑度が低く効率的な**5秒周期のポーリング(Polling)**方式を適用します。
- 管理者が該当のメニュータブに滞在している間のみ、5秒ごとにAPI（`GET /api/admin/metrics/system`）を呼び出し、画面の数値を同期させます。

### 2.3 UX/UI 設計 (MUI ベース)
- **Linear Progress Bar**: 単純な数値だけでなく、MUIの `<LinearProgress />` を使用して使用状況を視覚的に提供します。
  - CPU: Primary カラー (青系)
  - Memory: Info カラー (水色系)
  - Disk: Warning カラー (オレンジ/黄色系)
- **ローディングおよび例外処理**:
  - 初回アクセス時にはローディングスピナー（`<CircularProgress />`）を中央に表示します。
  - API通信に失敗した場合、画面のレイアウトを維持したまま、エラーメッセージ（"システムメトリクスの取得に失敗しました。"）を上部に表示します。
