# 機能仕様書: 監査ログダッシュボード (Audit Log Dashboard UI)

本文書は、`yoyaku_mate_admin` に実装された管理者操作の監査ログ確認画面の機能仕様を定義します。

> 作成日: 2026-07-22  
> 関連文書: [監査ログUI実装詳細書](../implementation/audit-log.md)

---

## 1. 概要 (Overview)

管理者がプラットフォーム上で実行したすべての主要な操作履歴（店舗承認・拒否等）をリアルタイムに視覚的に追跡・監査できる管理者専用画面です。

---

## 2. 画面レイアウトおよび構成要素 (UI Components)

1. **ヘッダーエリア**:
   - 画面タイトルおよび「すべての管理者操作の履歴 — リアルタイム自動更新（5秒間隔）」サブタイトル表示。
   - 手動更新アイコンボタン (`RefreshIcon`) による即時ログ再取得。
2. **フィルターエリア**:
   - アクション別ドロップダウンフィルター (`STORE_APPROVED`, `STORE_REJECTED`, `STORE_PENDING_REVIEW` 等)。
   - フィルタリング結果件数および全件数のリアルタイム表示。
3. **監査ログデータテーブル**:
   - `日時`: 日本ローカル時刻形式 (`toLocaleString('ja-JP')`) で表示。
   - `操作内容`: アクション種別に応じたカラー Chip（例: `STORE_APPROVED` は成功色, `STORE_REJECTED` はエラー色）。
   - `対象`: 操作対象の ID および説明。
   - `ステータス`: 処理結果 (`SUCCESS` / `FAILED`) Chip 表示。
   - `詳細`: 承認/拒否理由コメント等の要約表示。

---

## 3. リアルタイム更新仕様 (Real-time Behavior)

* **HTTP ポーリング**: 5秒ごとに `getAuditLogs()` API を非同期で呼び出し、画面を最新の状態に維持します。
* **クライアントフィルター**: API から取得したログデータに対して即時にアクションフィルターを適用します。
