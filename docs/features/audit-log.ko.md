# 기능 사양서: 감사 로그 대시보드 (Audit Log Dashboard UI)

본 문서는 `yoyaku_mate_admin`에 구현된 관리자 행위 감사 로그 확인 화면의 기능 사양을 정의합니다.

> 작성일: 2026-07-22  
> 관련 문서: [감사 로그 UI 구현 상세서](../implementation/audit-log.ko.md)

---

## 1. 개요 (Overview)

관리자가 플랫폼 상에서 실행한 모든 주요 작업 이력(가게 승인/반려 등)을 실시간으로 시각적 추적 및 감사할 수 있는 관리자 전용 화면입니다.

---

## 2. 화면 레이아웃 및 구성 요소 (UI Components)

1. **헤더 영역**:
   - 화면 타이틀 및 "모든 관리자 작업 이력 — 실시간 자동 갱신 (5초 간격)" 부타이틀 표시.
   - 수동 새로고침 아이콘 버튼 (`RefreshIcon`)을 통한 즉시 로그 재조회.
2. **필터 영역**:
   - 액션별 드롭다운 필터 (`STORE_APPROVED`, `STORE_REJECTED`, `STORE_PENDING_REVIEW` 등).
   - 필터링 결과 건수 및 전체 건수 실시간 표시.
3. **감사 로그 데이터 테이블**:
   - `日時` (일시): 일본 로컬 시간 포맷 (`toLocaleString('ja-JP')`)으로 표시.
   - `操作内容` (작업 내용): 액션 종류에 따른 컬러 Chip (예: `STORE_APPROVED`는 성공 색상, `STORE_REJECTED`는 에러 색상).
   - `対象` (대상): 작업 대상의 ID 및 설명.
   - `ステータス` (상태): 처리 결과 (`SUCCESS` / `FAILED`) Chip 표시.
   - `詳細` (상세): 승인/반려 사유 코멘트 등 요약 표시.

---

## 3. 실시간 갱신 사양 (Real-time Behavior)

* **HTTP 폴링**: 5초마다 `getAuditLogs()` API를 비동기로 호출하여 화면을 최신 상태로 유지합니다.
* **클라이언트 필터**: API로 받아온 로그 데이터에 대해 즉시 액션 필터를 적용합니다.
